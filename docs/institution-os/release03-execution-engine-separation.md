# Release 0.3 and Execution Engine separation

Release 0.3 is complete and operationally separate from the inactive Phase 4.2 Execution Engine. The P42-D001 equipment/access deferral is not a Release 0.3 defect, and it does not stop the non-Execution-Engine public operation.

## Continuation boundary

Release 0.3 and the public Citizen Audit operation may continue to be operated, promoted, expanded, and monetized without Phase 4.2 only while all of the following remain true:

* no untrusted validator code is executed;
* the Execution Engine remains inactive;
* no production-confinement, secure-sandbox, or Phase 4.2-completion claim is made;
* no automated execution result is represented as an institutional acceptance decision;
* human review remains required for research acceptance and publication;
* Issues #9 and #15 remain controlling, PR #21 remains unmerged, and all governed mutation/runtime-activation holds remain enforced.

## Allowed continuing work

The boundary permits public audit publication; manual and human-reviewed AI-assisted research; source discovery and verification; evidence-library expansion; financial and statutory analysis; contradiction/gap analysis; publication and report generation; public-website work unrelated to Execution Engine activation; audit-request intake; supporter, sponsorship, membership, consulting, and research-service revenue; non-executing prompts, schemas, methodologies, and research workflows; and controlled design documentation that does not begin prohibited Phase 4.2 implementation.

## Prohibited work

The boundary prohibits activation or deployment of the Execution Engine; execution of untrusted validator code; claims that the Execution Engine is production-ready or that a secure sandbox/OS confinement boundary exists; Phase 4.2 implementation; approval of P42-D001 or P42-D003; merging PR #21; bypassing Issues #9 or #15; self-installing or self-authorizing runtime capabilities; unattended autonomous repository mutation; and treating WSL, containers, chroots, CI, or static inspection as equivalent to required clean UEFI VM reproduction evidence.

This record preserves all existing holds. It does not authorize implementation, deployment, activation, merge, GitHub changes, Cloudflare changes, or a change to the approved P42-D002 object.
