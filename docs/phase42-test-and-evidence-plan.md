## Evidence classes and minimum acceptance set

Unit tests exercise parsing, categories, and deterministic policy decisions. Integration tests exercise orchestrator-supervisor interactions. Host-level tests observe processes, files, environment, and cleanup. Kernel-enforcement tests demonstrate the selected OS mechanisms actually deny or limit behavior. Platform-specific tests make no cross-platform inference. Independent reproduction repeats the approved procedure from a clean-room reconstruction.

Before Phase 4.2 could be considered for acceptance, the minimum set is: an approved decision record and threat model; exact commit/artifact and host provenance; successful and every failure-family evidence above for each claimed supported platform; unsupported-profile refusal evidence; direct negative evidence for filesystem, network, child process, inherited handles, identity/environment, bounded and safely staged input, and resource controls; supervisor-loss containment, restart reconciliation, orphan-tree detection, generation-safe process identity, process-tree, and residual-artifact checks for every terminal path; exact-byte provenance-to-execution evidence; staged-artifact expansion and path-safety evidence; mandatory-assurance-evidence and redaction review; final clean-tree confirmation; and an independent reproduction record. No platform may be claimed supported unless these applicable tests have been reproduced for that platform profile. **PROHIBITED CLAIM:** GitHub CI, a container build, or a single platform run certifies this package.
