## Evidence classes and minimum acceptance set

Unit tests exercise parsing, categories, and deterministic policy decisions. Integration tests exercise orchestrator-supervisor interactions. Host-level tests observe processes, files, environment, and cleanup. Kernel-enforcement tests demonstrate the selected OS mechanisms actually deny or limit behavior. Platform-specific tests make no cross-platform inference. Independent reproduction repeats the approved procedure from a clean-room reconstruction.

Before Phase 4.2 could be considered for acceptance, the minimum set is: an approved decision record and threat model; exact commit/artifact and host provenance; successful and every failure-family evidence above for each supported platform; direct negative evidence for filesystem, network, child process, identity/environment, and resource controls; process-tree and residual-artifact checks for every terminal path; audit/redaction review; final clean-tree confirmation; and an independent reproduction record. **PROHIBITED CLAIM:** GitHub CI, a container build, or a single platform run certifies this package.
