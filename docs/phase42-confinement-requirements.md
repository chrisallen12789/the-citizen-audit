# Phase 4.2 confinement requirements

## Status and use

**PLANNED.** These are stable, future acceptance requirements; none is implementation evidence. Candidate mechanisms are deliberately separated in [the decision matrix](phase42-platform-decision-matrix.md), and test evidence is defined in [the test and evidence plan](phase42-test-and-evidence-plan.md). Every requirement currently has status **PLANNED** unless its row says **OPEN** because a prerequisite decision or measurable value is unresolved.

## Interpretation

**REQUIRED:** acceptance must trace every applicable requirement to observable evidence in [the test plan](phase42-test-and-evidence-plan.md). **CANDIDATE:** mechanisms such as namespaces, cgroups, service identities, containers, or VMs can satisfy parts of these requirements only after review; none is selected here. **PROHIBITED CLAIM:** a passing JavaScript-realm or Node-worker test is not evidence that these OS-level requirements hold.
