# Phase 4.2 resource-budget framework

## Status

**PLANNED.** No numeric limit is approved or inferred here. Each value is **OPEN** and must be established empirically for the approved platform profile, workload, and enforcement mechanism. The supervisor contract uses these future budgets; see [the process-supervision contract](phase42-process-supervision-contract.md).

| Budget ID and category | Why it matters | Candidate measurement point | Candidate enforcement point | Failure response | Cleanup expectation | Observability requirement | OPEN value / decision |
| --- | --- | --- | --- | --- | --- | --- | --- |
| BUD-CPU / CPU allocation | Prevents sustained shared-host consumption. | Per-process/tree CPU counter. | cgroup, job/service control, or OS primitive. | Terminate/classify `RESOURCE_LIMIT`. | Terminate descendants and release policy handle. | CPU total and sampling source. | `P42-BUD-CPU`; P42-D008. |
| BUD-TIME / wall-clock duration | Stops noncooperative or stalled work. | Supervisor monotonic clock. | Supervisor deadline and tree termination. | Timeout category; no result acceptance. | Close transports and verify tree exit. | Start/deadline/terminal timestamps. | `P42-BUD-TIME`; P42-D009. |
| BUD-MEM / memory | Protects host availability. | Per-process/tree memory counter. | cgroup, job/service control, or OS primitive. | Terminate/classify limit breach. | Release tree and workspace. | Peak and enforcement event. | `P42-BUD-MEM`; P42-D008. |
| BUD-PROC / process count | Limits fork/spawn amplification. | Process tree census. | PID policy, cgroup, job/service control. | Prevent/contain and fail closed. | Verify no residual descendants. | Peak count and final census. | `P42-BUD-PROC`; P42-D010. |
| BUD-THREAD / thread count | Limits concurrency pressure where enforceable. | OS thread counter. | OS-specific primitive or documented non-enforcement. | Classify breach or refuse unsupported profile. | Ensure process termination. | Peak count and support status. | `P42-BUD-THREAD`; P42-D008. |
| BUD-FD / open files | Prevents descriptor exhaustion. | Per-process descriptor count. | rlimit, job/service, or equivalent. | Terminate/classify breach. | Close supervisor handles and verify process exit. | Peak count. | `P42-BUD-FD`; P42-D008. |
| BUD-FS / filesystem bytes | Bounds staged and validator-created data. | Workspace quota/accounting. | Filesystem quota, sparse workspace policy, or equivalent. | Stop/terminate and fail closed. | Remove workspace and verify absence. | Used/peak bytes and cleanup result. | `P42-BUD-FS`; P42-D006. |
| BUD-TEMP / temporary-storage bytes | Prevents scratch-space exhaustion and residue. | Workspace-only accounting. | Dedicated quota or bounded volume. | Stop/terminate and fail closed. | Erase owned temporary artifacts. | Used/peak bytes and residual scan. | `P42-BUD-TEMP`; P42-D006. |
| BUD-STDOUT / stdout bytes | Bounds diagnostic flooding. | Supervisor pipe byte counter. | Read limit, pipe policy, and termination. | Overflow category; do not parse as result. | Close capture and retain bounded record only. | Bytes, truncation, terminal category. | `P42-BUD-STDOUT`; P42-D012. |
| BUD-STDERR / stderr bytes | Bounds error-output flooding. | Supervisor pipe byte counter. | Read limit, pipe policy, and termination. | Overflow category; do not accept success. | Close capture and retain bounded record only. | Bytes, truncation, terminal category. | `P42-BUD-STDERR`; P42-D012. |
| BUD-RESULT / structured-result bytes | Protects parser and result integrity. | Result transport byte counter. | Framing/schema limit at supervisor boundary. | Reject result and terminate if required. | Discard untrusted payload safely. | Bytes, parse result, schema version. | `P42-BUD-RESULT`; P42-D012. |
| BUD-IPC / IPC message count | Prevents message amplification. | Supervisor transport counter. | Framed-channel count limit. | Stop channel, classify breach, fail closed. | Close IPC handles. | Count and final transport state. | `P42-BUD-IPC`; P42-D012. |
| BUD-RETRY / startup retries | Avoids repeated launch amplification. | Supervisor attempt counter. | Orchestrator/supervisor policy. | Stop retries and report deterministic category. | Fresh workspace for each permitted retry. | Attempt count and reason. | `P42-BUD-RETRY`; P42-D009. |
| BUD-GRACE / termination grace period | Balances orderly shutdown and host recovery. | Supervisor monotonic clock. | Supervisor escalation policy. | Force tree termination at expiry. | Verify residual-process absence. | Grace start/end and action record. | `P42-BUD-GRACE`; P42-D009. |

**REQUIRED:** an approved profile defines whether each enforcement point is hard, advisory, unavailable, or emulated, and refuses profiles that cannot meet mandatory requirements. **CANDIDATE:** cgroups, rlimits, job controls, service managers, and VM allocation are mechanisms to evaluate, not selected solutions. **PROHIBITED CLAIM:** an observed counter without an enforced and verified response proves a budget.
