Audit 002

Data Catalog

This catalog explains the machine-readable files behind Volume II. The data is meant to support review, not replace judgment.

DownloadsEvidence MapSource GateDecision Log

Available data files

CSV · public register

Claim database

Twenty-one claim records with claim text, section, source IDs, confidence, status, limitations, verification, and use notes.

CSV · archive gate

Claim archive traceability

Maps each claim to its cited source IDs and archive-gate status.

JSON · manifest

Publication manifest

Version, decision status, gate status, counts, graph schema, and explicit limitations.

CSV · evidence summary

Evidence by source

Counts how many evidence items were extracted from each source ID.

CSV · evidence summary

Evidence by category

Counts extracted evidence items by category: statistical, definitional, appropriation, disagreement, unresolved, and other classes.

How to use these files

  1. Start with the claim database to identify the claim ID.
  2. Use claim archive traceability to see whether the cited source trail cleared the archive gate.
  3. Use evidence summaries to see which sources carry the most evidentiary weight.
  4. Read the limitations before turning a claim into a public statement.
  5. Use the Decision Log to understand why a figure, claim, source, or passage was included, excluded, quarantined, or narrowed.

What these files do not prove

A CSV row does not prove a public claim by itself. The row is a pointer. A reviewer still has to read the source, check the cited basis, and respect the limitation. The catalog exists to make that process faster and more transparent.
