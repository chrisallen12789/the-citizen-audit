# Kernel Scaling Doctrine

Status: foundational architecture

The Citizen Audit may eventually operate with very large numbers of agents. The target is not a swarm of unconstrained artificial decision-makers. The target is a disciplined institutional workforce made of many narrow, auditable, replaceable agents.

## Scaling principle

Scale through specialization, not broad authority.

A future system with 100,000 agents should behave more like a transparent civil service than an uncontrolled collective intelligence.

## Agent hierarchy

```text
Institution Kernel
  Executive Orchestrator
    Department Coordinators
      Team Coordinators
        Worker Agents
```

## Worker-agent pattern

Most agents should do one narrow task:

- check one URL;
- compare one file;
- classify one challenge;
- compute one checksum;
- validate one schema;
- inspect one source record;
- extract one table;
- flag one inconsistency;
- write one report section.

## Coordinator-agent pattern

Coordinators do not decide truth. They route work, aggregate reports, enforce dependency order, and escalate unresolved findings.

## Department model

Initial departments:

- Engineering;
- Evidence;
- Archives;
- Research;
- Quality;
- Institutional Governance;
- Release Management;
- Public Challenge Intake.

## Failure model

At large scale, failure is normal. The kernel must assume agents will fail, disagree, duplicate work, or produce incomplete output.

The solution is not trust. The solution is:

- logs;
- retries;
- independent checks;
- permission limits;
- report aggregation;
- human approval for truth-bearing changes;
- permanent records of unresolved failures.

## Bias control

Founder bias, contributor bias, model bias, and public pressure must be treated as operational risks.

The institution reduces bias through:

- public rules;
- reproducible evidence;
- explicit unknowns;
- decision logs;
- challenge records;
- constrained agents;
- immutable identifiers;
- human approval gates;
- independent verification paths.

## Growth doctrine

Do not add agents for novelty. Add agents when a recurring institutional responsibility can be made safer, faster, more consistent, or more transparent.

Every new agent must declare:

- mission;
- authority level;
- allowed actions;
- forbidden actions;
- input records;
- output records;
- report path;
- escalation rules.

## End state

The Citizen Audit should become a public audit institution that can scale far beyond the labor capacity of one founder while reducing, not increasing, the risk of arbitrary authority.
