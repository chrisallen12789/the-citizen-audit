# Institutional Runtime

Status: living operational state

This file records the current operating context for The Citizen Audit. Unlike `institution/ai-bootstrap.md`, this file is expected to change as the institution evolves.

## Current repository

`https://github.com/chrisallen12789/the-citizen-audit`

## Current operating model

The Citizen Audit is evolving from a publication platform into an Institution Operating System.

The current live site remains stable while Platform v2 and the institutional kernel evolve behind it.

The software architecture is organized around five major systems:

```text
Institution
Kernel
Memory
Workforce
Platform
```

## Current institutional layer

The institutional layer includes:

- Public Charter
- Constitution
- Contributor Oath
- Citizen Standard
- Unknowns Register
- Operating Principles
- Vision
- 2035 Long-Term Vision
- AI Bootstrap

## Current kernel layer

The kernel currently includes:

- agent runtime
- event logging
- executive status
- permission rules
- authority levels
- Constitution Guardian gate
- agent registry

The next kernel objective is to move from descriptive authority declarations toward executable authority checks.

## Current workforce layer

Current active agents include:

- Constitution Guardian
- Repair Agent
- ADS Steward
- Source Steward
- Publication Builder
- Release Steward
- Content Hygiene

Planned agents include:

- Challenge Intake Agent

Additional agents are expected.

The long-term goal is many specialized agents organized into departments with narrow authority and explicit permission boundaries.

## Current memory layer

Memory Engine v1 exists.

It contains:

- node schema
- edge schema
- graph
- memory status
- memory query
- memory export

Institutional memory is represented as a graph of nodes and relationships.

Memory records relationships.

Memory does not create conclusions.

## Current proposal layer

Proposal Engine v1 exists.

Institutional improvements are represented as proposals rather than automatic changes.

Agents recommend.

Humans approve.

## Current publication layer

Platform v2 is being developed around the Audit Definition Specification.

Current publication infrastructure includes:

- structured data model
- ADS export
- ADS validation
- evidence graph generation
- search output generation
- trace records
- detail renderers
- QA gates
- release-candidate generation

## Operating constraint

No subsystem may claim institutional completion unless the repository contains the corresponding record.

Attestation is not a record.

A gate cannot be passed by assertion.
